## Проблема
После завершения деплоя «светофор» показывает «нет данных» вместо зелёного. Причина в логике на `src/pages/AdminSystemSettings.tsx:230-243`:

- Зелёный загорается только при `last.status === "deployed" && app.status === "deployed"`.
- Если массив `deploys` пуст / последний деплой имеет статус-алиас (`success`, `active`, `online`, `running`) / `app.status` имеет другое написание — попадаем в `default` → «нет данных».
- Пока `deployStatus === null` (первая загрузка/ошибка сети) — тоже «нет данных», без отличия от реального состояния.

## Что изменить (только UI-логика)

Файл: `src/pages/AdminSystemSettings.tsx`, функция-IIFE индикатора (строки 230–250).

1. Расширить набор «успешных» статусов: `deployed`, `success`, `active`, `online`, `running`, `ok`.
2. Зелёный загорается, если **либо** `app.status` — успешный, **либо** последний деплой — успешный (не требовать одновременно оба).
3. Расширить «идёт деплой»: `deploy`, `building`, `pending`, `queued`, `in_progress`, `processing`.
4. Расширить «ошибка»: `failed`, `stopped`, `error`, `cancelled`, `timeout`.
5. Различать три состояния «нет данных»:
   - `deployStatusLoading && !deployStatus` → серый + «загрузка…»
   - `!deployStatus && !deployStatusLoading` → серый + «нет связи с Timeweb»
   - `deployStatus` есть, но статус неизвестен → жёлтый + сырая строка статуса (для диагностики).
6. Добавить `title` с сырыми значениями `app.status` / `last.status` — чтобы при наведении видеть, что вернул Timeweb (поможет, если появятся новые алиасы).
