## Правило автодеплоя

Зафиксировать в памяти проекта (`mem://index.md` → Core) единое правило для всех будущих сессий:

> **Авто-деплой.** Backend (Edge Functions, миграции) деплоится автоматически — действий не требуется. После любых правок фронтенда в рамках ответа агент **один раз в самом конце** вызывает `preview_ui--publish` — даже для мелких правок (типографика, цвет, одна строка текста). В сообщении явно сообщает: «Публикую — деплой пойдёт ~через минуту».

## Как это будет работать на практике

1. В течение ответа агент копит все правки фронта (несколько файлов, несколько итераций) — без промежуточных публикаций.
2. Если в ответе менялся только бэкенд (Edge Function/SQL) — публикация не запускается, бэкенд деплоится сам.
3. В конце ответа, если был тронут хотя бы один файл фронта, агент:
   - проверяет `security--get_scan_results` (требование `preview_ui--publish`);
   - вызывает `preview_ui--publish` с актуальными `website_info_status` / `website_info_summary`;
   - сообщает «деплой запущен, будет live через ~1 минуту» (на Timeweb — дольше, по вашему GitHub-флоу).
4. Если security-сканер блокирует публикацию (есть critical-findings) — агент **не** публикует, а сообщает об этом и ждёт решения.
5. Если публикация технически провалилась (например, DDoS на Timeweb, как было недавно) — агент сообщает об ошибке и НЕ повторяет попытку в цикле.

## Что НЕ меняется

- Видимость публикации (public/private) и кастомные домены — только по явному запросу.
- Slug проекта — не меняется.
- Это правило не отменяет вашу возможность в любой момент сказать «не публикуй сейчас» — приоритет всегда у явной команды в текущем сообщении.

## Технические детали

- Запись в память: добавить строку в раздел `## Core` файла `mem://index.md`:
  `Auto-publish frontend once at end of every reply via preview_ui--publish (even for tiny edits). Backend deploys itself.`
- Никаких изменений в коде проекта, конфигах Vite, GitHub Actions или Supabase не требуется — это поведенческое правило агента.
- Первая «боевая» автопубликация произойдёт в конце следующего ответа, где будут реальные правки фронта.
